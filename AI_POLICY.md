# AI Policy

I (the creator of catoscript) am not against AI, but I am against AI slop and laziness passed off as "AI-assistance". This doc is not a stance on AI, it's simply an honest account of how it is and is not used in this repo.

**catoscript is built by a real human bean.** The language design, the architecture, and every line of Kotlin under `src/main/kotlin/com/catoscript/` is handrcafted and owned by a human. This repo is not AI generated; the maintainer is responsible for all of it.

**AI is used in this repo for the following:** documentation drafting, commit messages, (basically all of my documentation and commit messages are AI generated from a wall of spghetti text/thoughts I give ai and ask it to turn it into something more coherent and structured) mechanical edits (large renames, consistency checks, scaffolding). AI handles things like filler and unremarkable stuff that typically eats hours of time and attention away from actual work.

**Contributors READ THIS**: code you submit must be YOUR code that you can fully explain, can place inside catoscript's bigger picture, and conforms to the rules and philosophy of catoscript as defined in repo documentation. Documentation may be drafted, sorted, scaffolded, or otherwise AI assisted, but design decisions behind it MUST be yours. Pull requests that are simply AI generated code with the human acting as reviewer rather than author will be declined. If you are unsure whether a change counts as code or docs for this policy, please ask first before submitting anything.
